# X投稿クリエイター

X（旧Twitter）でバズる投稿を生成する。引数 `$ARGUMENTS` にテーマが渡された場合はそれを元に直接生成する。

## 手順

### 1. ネタ確認

`$ARGUMENTS` が空の場合のみ確認する。すでにテーマがあれば手順2へ進む。

- 投稿したいテーマや出来事をユーザーに確認する
- 決まっていなければ、以下から3つ候補を提案する：
  - 最近の気づき・学び
  - 業務・経営での出来事
  - 業界トレンドへの意見

### 2. 投稿タイプを選択（迷っていれば内容に合ったものを推薦）

| タイプ | パターン例 |
|---|---|
| 気づき・学び系 | 「〇〇して気づいたこと」 |
| 逆説・意外性系 | 「実は〇〇は間違いだった」 |
| リスト系 | 「〇〇な人の3つの特徴」 |
| 問いかけ系 | 「あなたは〇〇できていますか？」 |
| ストーリー系 | 「〇〇だった私が△△になれた理由」 |
| データ・事実系 | 「〇〇の意外な数字」 |

### 3. 投稿3案を生成

以下の形式で出力する。各案は**140字以内**（日本語換算）：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
案1｜インパクト重視
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（本文）

💡 狙い: （バズる理由を一言）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
案2｜共感重視
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（本文）

💡 狙い: （バズる理由を一言）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
案3｜情報価値重視
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（本文）

💡 狙い: （バズる理由を一言）
```

生成後、スレッド展開・トーン調整・字数変更などのリクエストに対応する。

### 4. ファイル保存

採用した投稿案を `company/CMO/campaigns/posts/YYYY-MM-DD_テーマ.md` として保存する。
ファイル内に3案すべてと採用案の記録を残す。

## バズる投稿の原則（生成時に必ず守る）

- **1行目が命**: スクロールを止める冒頭の一文を最優先で磨く
- **具体的な数字**: 「多い」ではなく「3倍」「87%」
- **感情を動かす**: 驚き・共感・反論・学びのどれかを必ず狙う
- **短く切る**: 1文を短く、改行を多用して読みやすくする
- **引用されやすい一文**: 誰かが「これ！」とRTしたくなる言葉を入れる

## 注意事項

- 姜大成（CEO）として発信する場合は `company/` 配下のcontextを参照し、ビズリンクの実績・ビジョン・価値観を反映する
- 誇張・虚偽・炎上リスクのある表現は避ける
- ハッシュタグは求められた場合のみ提案する（多用しない）
