# Note Writer

note.com向けの記事コンテンツを自動生成するスキル。

## When to use

ユーザーがnoteの記事を書きたい、記事のネタを形にしたい、と言ったとき。

## Instructions

1. **テーマ・目的の確認**
   - 記事のテーマや伝えたいことをユーザーに確認する
   - まだ決まっていない場合は、最近の業務・関心事・経験から候補を3つ提案する

2. **ターゲット読者の設定**
   - 誰に向けた記事か確認する（例: 経営者、エンジニア、就活生、一般読者）
   - 読者の悩みや知りたいことを明確にする

3. **記事フォーマットの選択**
   以下から選んでもらう（迷っていれば推薦する）：
   - **ノウハウ系**: 「〇〇するための5つの方法」
   - **体験談系**: 「〇〇してみた結果」
   - **考察系**: 「〇〇について本気で考えてみた」
   - **まとめ系**: 「〇〇を徹底解説」

4. **記事を生成する**
   以下の構成で出力する：

   ```
   【タイトル】
   （読者の興味を引く、具体的で検索されやすいタイトル）

   【リード文】
   （100〜150字。読者の共感を引き、記事を読む理由を作る）

   【本文】
   ## 見出し1
   （本文）

   ## 見出し2
   （本文）

   ## 見出し3
   （本文）

   【まとめ】
   （要点の整理 + 読者へのメッセージ）

   【ハッシュタグ】
   #タグ1 #タグ2 #タグ3
   ```

5. **ファイル保存**
   - 記事を `company/CMO/campaigns/notes/YYYY-MM-DD_タイトル.md` として保存する
   - ファイル名の日付は実行日、タイトルは記事タイトルを短縮したものを使用する

6. **調整対応**
   - 生成後、トーン（堅い/柔らかい）、文字数、構成の変更リクエストに対応する
   - 必要に応じてタイトル案を複数パターン提示する

## 注意事項

- 株式会社ビズリンクのCEO姜大成の視点・経験に基づいた記事を書く場合は、`company/` 配下のcontextを参照してパーソナリティを反映する
- 読者に価値を届けることを最優先に、宣伝色は控えめにする
- note特有の文体（話しかけるような口調、適度な改行）を意識する
