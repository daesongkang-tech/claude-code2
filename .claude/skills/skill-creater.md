# スキルクリエイター

新しいClaude Codeスキルを対話形式で作成する。引数にスキル名が渡された場合（`$ARGUMENTS`）はそれを初期値として使う。

## 手順

以下の質問を順番にユーザーに確認する。各回答を得てから次に進む。

### 1. スキル名の確認

- スキル名 = スラッシュコマンド名（例: `deploy` → `/deploy`）
- 引数が渡されている場合: `$ARGUMENTS` をデフォルト案として提示し確認する
- 命名規則: 小文字英数字とハイフンのみ（最大64文字）

### 2. 目的・説明文の作成

- このスキルが何をするかを一文で確認する
- **重要**: `description` はClaudeが自動発動するかどうかを判断するために使う
  - 自動発動させたい場合 → ユーザーが自然に言いそうなキーワードを含める
  - 手動のみにしたい場合 → `disable-model-invocation: true` を設定する

### 3. 呼び出し制御の確認

以下の2択をユーザーに選ばせる：

- **手動のみ** (`disable-model-invocation: true`): デプロイ・コミット・送信など副作用のある操作
- **自動＋手動** (デフォルト): コード説明・調査・提案など安全な操作

### 4. 引数の有無を確認

- 引数が必要か確認する（例: `/fix-issue 123` のように）
- 必要な場合は `argument-hint` と `$ARGUMENTS` の使い方を説明してから組み込む

### 5. スキルの動作内容を定義

- Claude が実行すべき具体的な手順を番号付きで確認する

### 6. スキルファイルを作成

収集した情報をもとに、以下の形式でスキルファイルを作成する。

**ファイルパス**: `.claude/skills/<スキル名>.md`（プロジェクトレベル）

```markdown
---
name: <スキル名>
description: <Claudeが自動判断に使う説明文。いつ・どんな言葉で発動するかを記述>
disable-model-invocation: true  # 手動のみの場合のみ記載
argument-hint: [<引数のヒント>]  # 引数がある場合のみ記載
---

# <スキル名（人間が読みやすい形式）>

<スキルの概要説明>

## 手順

1. <ステップ1>
2. <ステップ2>
3. <ステップ3>
```

### 7. 完了報告

作成後、ユーザーに以下を伝える：
- 作成したファイルパス
- 呼び出し方: `/<スキル名>` または `/<スキル名> <引数>`
- `disable-model-invocation` の設定内容と意味
- このセッションからすぐに有効になること

## フロントマター参照

| フィールド | 説明 |
|---|---|
| `name` | スラッシュコマンド名（省略時はファイル名を使用） |
| `description` | Claudeが自動発動判断に使う説明（**推奨**） |
| `disable-model-invocation: true` | Claudeによる自動発動を無効化（手動専用） |
| `user-invocable: false` | ユーザーメニューから非表示（Claude専用） |
| `argument-hint` | オートコンプリートに表示する引数ヒント |
| `allowed-tools` | このスキル実行中に許可するツール |
| `context: fork` | サブエージェントの独立コンテキストで実行 |
